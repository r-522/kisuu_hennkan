# 実装依頼プロンプト

あなたはフロントエンドエンジニアです。以下の要件で、基数変換を学習できる四択クイズサイトを作成してください。

## 目的

- 基数変換を反復練習できる静的 Web アプリを作成する。
- 利用者が「ア・イ・ウ・エ」の四択で回答できるようにする。
- 1問ごとに解説を表示し、考え方と別解を学べるようにする。
- エンドレスに次の問題へ進めるようにする。

## 技術要件

- HTML と JavaScript のみで作成する。
- フレームワークは使用しない。
- TypeScript は使用しない。
- DB は使用しない。
- API サーバーは使用しない。
- 外部 CDN は使用しない。
- 画像アセットは使用しない。
- 必要なスタイルは `index.html` 内の `style` タグに記述する。
- JavaScript は `app.js` に分離する。
- Vercel で静的サイトとしてデプロイできる構成にする。
- ビルド工程なしで動作するようにする。

## 作成ファイル

- `index.html`
  - 画面構造
  - 最低限のスタイル
  - `app.js` の読み込み
- `app.js`
  - 問題生成
  - 選択肢生成
  - 回答判定
  - 解説生成
  - スコア管理
  - 次問生成

## クイズ仕様

- 問題は基数変換とする。
- 対応する基数は以下を基本とする。
  - 2進数
  - 8進数
  - 10進数
  - 16進数
- 出題例
  - `1011₂` を `10進数` に変換
  - `255₁₀` を `16進数` に変換
  - `777₈` を `2進数` に変換
  - `3A₁₆` を `10進数` に変換
- 変換元の基数と変換先の基数は毎回ランダムにする。
- 変換元と変換先が同じにならないようにする。
- 内部的には整数値を基準にして問題を生成する。
- 難しすぎないように、初期値は `1` から `4095` 程度の範囲にする。
- 問題ごとに正解を1つ、誤答を3つ生成する。
- 選択肢は必ず4つ表示する。
- 選択肢ラベルは必ず以下にする。
  - ア
  - イ
  - ウ
  - エ
- 選択肢の値は重複させない。
- 正解位置は毎回ランダムにする。
- 回答後は選択肢を無効化する。
- 回答後に正誤を表示する。
- 回答後に「次の問題」ボタンを表示する。
- 「次の問題」を押すと新しい問題を生成する。
- 問題数に上限を設けず、エンドレスに回答できるようにする。

## 解説仕様

- 回答後に必ず解説を表示する。
- 解説には以下を含める。
  - 正解
  - 考え方
  - 計算手順
  - 別解
- 考え方では、変換元の数を10進数へ直す過程を説明する。
- 10進数から変換先へ直す場合は、割り算と余りを使った手順を表示する。
- 2進数・8進数・16進数の相互変換では、可能な場合に桁のグルーピングによる別解を表示する。
- 例
  - 2進数から8進数は3ビットごとに区切る。
  - 2進数から16進数は4ビットごとに区切る。
  - 8進数から16進数は一度2進数に直してから4ビットごとに区切る。
- 別解が自然に作れない組み合わせでは、10進数を経由する方法を別表現で説明する。

## 画面仕様

- 画面上部にタイトルを表示する。
- 現在の問題を大きく表示する。
- 4つの選択肢をボタンで表示する。
- スコアを表示する。
  - 正解数
  - 回答数
  - 正答率
  - 連続正解数
- 結果表示エリアを用意する。
- 解説表示エリアを用意する。
- リセットボタンを用意する。
- レスポンシブ対応にする。
- スマートフォンでも使いやすいボタンサイズにする。

## アクセシビリティ

- ボタンはキーボード操作できるようにする。
- `aria-live` を使用して正誤結果を読み上げ対象にする。
- 色だけに依存せず、テキストでも正誤を示す。
- コントラストを確保する。
- フォーカス表示を消さない。

## 実装方針

- 主要関数を分けて実装する。
- 推奨する関数例
  - `generateQuestion()`
  - `generateChoices(correctAnswer, targetBase)`
  - `toBaseString(value, base)`
  - `fromBaseString(text, base)`
  - `shuffle(array)`
  - `renderQuestion(question)`
  - `handleAnswer(choice)`
  - `buildExplanation(question)`
  - `buildAlternativeExplanation(question)`
  - `updateScore(isCorrect)`
  - `resetQuiz()`
- 数値表記は `0-9A-Z` を使用する。
- 16進数は大文字に統一する。
- `eval` は使用しない。
- グローバル変数は最小限にする。
- DOM 参照は初期化時にまとめる。
- 同じロジックを重複させない。

## 誤答生成

- 誤答は正解に近い値を中心に作る。
- 例
  - 正解値に `±1`
  - 正解値に `±base`
  - 桁の読み間違い
  - 変換元基数を誤って10進数として扱った値
- 誤答は変換先の基数表記に変換して表示する。
- 誤答が正解と同じ場合は再生成する。
- 選択肢が4つ揃うまで生成する。

## 品質要件

- ブラウザで直接 `index.html` を開いて動作する。
- Vercel の静的ホスティングで動作する。
- コンソールエラーが出ない。
- スマートフォン幅でも崩れない。
- 連続で何問回答しても状態が破綻しない。
- 同じ選択肢が重複しない。
- 正解が必ず選択肢に含まれる。
- 回答後に必ず解説が表示される。

## 出力形式

- 完成版の `index.html` と `app.js` を出力する。
- 各ファイルはファイル名が分かる形で分ける。
- 省略せず、実行可能な完全なコードを出力する。
