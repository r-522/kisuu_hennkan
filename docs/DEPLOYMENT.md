\# デプロイ手順



\## 前提



\- 静的サイトとしてデプロイする。

\- ビルド工程は不要。

\- DB は不要。

\- サーバー API は不要。



\## 必要ファイル



\- `index.html`

\- `app.js`



\## ローカル確認



\- ブラウザで直接確認する場合

&#x20; - `index.html` を開く。

\- ローカルサーバーで確認する場合

&#x20; - `python3 -m http.server 8000`

&#x20; - ブラウザで `http://localhost:8000` を開く。



\## Vercel での設定



\- Framework Preset

&#x20; - `Other`

\- Build Command

&#x20; - 空欄

\- Output Directory

&#x20; - `.`

\- Install Command

&#x20; - 空欄

\- Root Directory

&#x20; - リポジトリルート



\## GitHub 経由のデプロイ



\- リポジトリを作成する。

\- `index.html` と `app.js` をルートに配置する。

\- 必要な Markdown ドキュメントを配置する。

\- Vercel でリポジトリをインポートする。

\- 設定を確認してデプロイする。

\- デプロイ後のページでクイズが表示されることを確認する。



\## デプロイ後確認



\- ページが表示される。

\- 1問目が自動生成される。

\- `ア・イ・ウ・エ` の選択肢が表示される。

\- 回答できる。

\- 正誤が表示される。

\- 解説が表示される。

\- 次の問題へ進める。

\- 複数回回答してもエラーが出ない。

\- スマートフォン幅で崩れない。



\## 注意点



\- `package.json` は不要。

\- `vercel.json` は原則不要。

\- 外部ライブラリを追加しない。

\- Vercel のビルド設定で不要なコマンドを指定しない。
